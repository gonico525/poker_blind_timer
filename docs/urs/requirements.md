# ノーリミットホールデム ブラインドタイマー 要求仕様書

## 1. プロジェクト概要

### 1.1 目的
ノーリミットホールデムトーナメント用のブラインドタイマーをブラウザベースで提供し、ローカル環境で完結する直感的なツールを開発する。

### 1.2 対象ユーザー
- ポーカートーナメントの主催者・ディーラー
- ホームゲームの運営者

### 1.3 動作環境
- **プラットフォーム**: Webブラウザ（PC向け）
- **実行環境**: ローカル環境で完結（サーバー不要）
- **対応ブラウザ**: モダンブラウザ（Chrome, Firefox, Safari, Edge）
- **表示デバイス**: PCモニター

---

## 2. 機能要件

### 2.1 ブラインド構造管理

#### 2.1.1 ブラインドレベル設定
- **レベル数**: 無制限に追加可能
- **デフォルト設定**: 10レベル
- **設定項目**:
  - スモールブラインド（SB）
  - ビッグブラインド（BB）
  - アンティ（レベル1から設定可能）

#### 2.1.2 カスタマイズ機能
- ブラインド金額の自由編集
- レベルの追加・削除
- レベル順序の変更

### 2.2 時間管理

#### 2.2.1 タイマー設定
- **レベル時間**: 全レベル共通の時間設定（分単位）
- **カスタマイズ**: レベル時間を変更可能
- **初期値**: 10分（調整可能）

#### 2.2.2 タイマー操作
- **開始/停止**: タイマーの開始と停止
- **一時停止/再開**: タイマーの一時停止と再開
- **リセット**: タイマーを初期状態に戻す

#### 2.2.3 レベル操作
- **自動進行**: 時間終了時に次のレベルへ自動遷移
- **手動変更**: 前後のレベルへ手動でジャンプ可能
- **レベル戻し**: 前のレベルに戻る機能

### 2.3 表示機能

#### 2.3.1 基本表示情報
- **現在のレベル情報**:
  - レベル番号
  - スモールブラインド（SB）
  - ビッグブラインド（BB）
  - アンティ
- **次のレベル情報**:
  - 次レベルのSB/BB/アンティ
- **時間情報**:
  - 現在レベルの残り時間（MM:SS形式）

#### 2.3.2 表示モード
- **タイマー表示モード**:
  - 大きな文字表示で視認性を最大化
  - 現在のブラインド情報を強調
  - フルスクリーン表示対応
- **設定/管理モード**:
  - ブラインド構造の編集
  - プリセット管理
  - 各種設定

### 2.4 通知機能

#### 2.4.1 音声通知
- **レベル変更時**: 新しいレベル開始の通知音
- **警告音**: レベル終了前の警告（例：残り1分、30秒）
- **音量調整**: 音量のオン/オフ、レベル調整

### 2.5 データ管理

#### 2.5.1 保存機能
- **ローカルストレージ**: ブラウザのlocalStorageを使用
- **保存対象**:
  - カスタムブラインド構造
  - プリセット設定
  - タイマー設定（レベル時間）
  - UI設定（ダークモード等）

#### 2.5.2 プリセット管理
- **プリセット作成**: カスタム構造を名前付きで保存
- **プリセット選択**: 保存済みプリセットの読み込み
- **プリセット編集**: 既存プリセットの変更
- **プリセット削除**: 不要なプリセットの削除
- **デフォルトプリセット**:
  - スタンダードトーナメント（例：25/50 → 50/100 → ...）
  - ターボトーナメント
  - ディープスタック

### 2.6 インポート/エクスポート

#### 2.6.1 データエクスポート
- **形式**: JSON形式
- **内容**: ブラインド構造、設定情報
- **用途**: バックアップ、他環境への移行

#### 2.6.2 データインポート
- **形式**: JSON形式
- **検証**: インポート時のデータ妥当性チェック

## 3. 非機能要件

### 3.1 ユーザビリティ（UX）

#### 3.1.1 直感的操作
- 最小限のクリック数で主要操作を完了
- 明確なボタン配置とラベリング
- 操作に対する即座のフィードバック

#### 3.1.2 視認性
- タイマー表示時の大きく読みやすいフォント
- 高コントラストの色使い
- レベル変更時の視覚的フィードバック

#### 3.1.3 レスポンシブデザイン
- PC画面サイズに最適化
- フルスクリーン表示対応
- 最小解像度: 1280x720px

### 3.2 デザイン

#### 3.2.1 テーマ
- **スタイル**: シンプルでポーカーっぽいかっこいいデザイン
- **カラーパレット**:
  - ダークモード: ポーカーテーブルを連想させる緑、黒、ゴールドアクセント
  - ライトモード: 白、グレー、アクセントカラー
- **モード切り替え**: ダークモード/ライトモードの切り替え機能

#### 3.2.2 タイポグラフィ
- **タイマー表示**: 太字、大きなフォントサイズ（100px以上）
- **フォント**: 可読性の高いサンセリフフォント
- **数字**: 等幅フォントでブレを防止

### 3.3 パフォーマンス
- **タイマー精度**: 1秒単位で正確な時間管理
- **動作**: スムーズなアニメーション（60fps目標）
- **起動時間**: 1秒以内のロード完了

### 3.4 互換性
- **ブラウザ**: 最新版および1世代前のメジャーブラウザ対応
- **データ永続化**: localStorage対応ブラウザ

### 3.5 セキュリティ
- **データ保存**: 全てローカルストレージ（外部送信なし）
- **プライバシー**: ユーザーデータの収集なし

## 4. システム構成

### 4.1 技術スタック（推奨）
- **フロントエンド**: HTML5, CSS3, JavaScript（Vanilla or React/Vue等）
- **データ保存**: localStorage API
- **音声**: Web Audio API / HTML5 Audio

### 4.2 ファイル構成
- **Single Page Application (SPA)**: 単一HTMLファイル + JS/CSS
- **オフライン動作**: 全リソースをローカルに含む

## 5. ユーザーストーリー

### 5.1 トーナメント開始
```
トーナメント主催者として
事前に設定したプリセットを選択し
ワンクリックでタイマーを開始したい
```

### 5.2 ブラインド構造のカスタマイズ
```
ホームゲーム運営者として
独自のブラインド構造を作成・保存し
次回以降も簡単に再利用したい
```

### 5.3 タイマー操作
```
ディーラーとして
トラブル発生時にタイマーを一時停止し
問題解決後にスムーズに再開したい
```

### 5.4 視認性
```
全プレイヤーとして
離れた位置からでも現在のブラインド情報を
明確に確認したい
```

## 7. 制約事項

### 7.1 初期バージョンの制限
- 休憩時間機能は含まない（将来的に追加予定）
- レベルごとの個別時間設定は不可（全レベル共通）
- プロジェクター表示への最適化は行わない

### 7.2 技術的制約
- インターネット接続は不要
- サーバーサイド処理なし（完全クライアントサイド）

## 9. 用語集

| 用語 | 説明 |
|------|------|
| ブラインド | 強制ベット。スモールブラインド（SB）とビッグブラインド（BB）がある |
| アンティ | 全プレイヤーが支払う強制ベット |
| レベル | ブラインド構造の段階。時間経過で上昇する |
| プリセット | 保存されたブラインド構造のテンプレート |
